**raw trip/city data in S3 в†’ progressively cleaned & validated в†’ modeled into a reporting-ready star schema**

...so business teams can answer questions like:

## рџҐ‰рџҐ€рџҐ‡ Medallion Layers

### Bronze вЂ” Raw Ingestion
Data lands as-is from S3, with metadata columns (`file_name`, `ingest_datetime`) added for traceability вЂ” no transformation logic here.

| File | Description |
|---|---|
| `bronze/city.py` | Batch load of city reference data via materialized view |
| `bronze/trips.py` | Streaming ingestion of trip data using **Auto Loader**, with schema evolution support |

### Silver вЂ” Cleaned & Validated
Data is standardized, validated, and de-duplicated.

| File | Description |
|---|---|
| `silver/calendar.py` | Generated date dimension (year/month/quarter/weekday/Indian holidays) |
| `silver/city.py` | Cleaned city dimension with standardized column names |
| `silver/trips.py` | Data quality expectations (valid dates, valid ratings) + **CDC-based upsert (SCD Type 1)** to prevent duplicate/stale trip records |

### Gold вЂ” Business-Ready
Fact and dimension tables are joined into a single, reporting-ready **star schema**.

| File | Description |
|---|---|
| `gold/trips_gold.sql` | Central `fact_trips` table вЂ” joins trips with city and calendar dimensions |
| `gold/city_views/*.sql` | Per-city filtered views for scoped regional reporting |

## вљ™пёЏ How to Run This Project

1. Clone this repository.
2. In Databricks: **Workspace в†’ Repos в†’ Add Repo**, and connect this GitHub repository.
3. Set up an **External Location** and **Storage Credential** in Unity Catalog pointing to your own S3 bucket.
4. Update `SOURCE_PATH` in `bronze/city.py` and `bronze/trips.py` to point to your bucket.
5. Create a new **Lakeflow Declarative Pipeline**, using `2. codes/bronze` and `2. codes/silver` as source folders.
6. Run the pipeline вЂ” Bronze, Silver, and Gold tables are created automatically under the `transportation` catalog.
7. Build a dashboard or connect Genie on top of `transportation.gold.fact_trips` to start querying.

> **Note:** This repo doesn't include real credentials or a live S3 bucket. Point `SOURCE_PATH` to your own bucket before running.
